## OpenAI Built-in Tools - Tool Call and Response Representation

| Tool | Tool Call Object | Tool Response Object |
|------|------------------|----------------------|
| Web Search | [`web_search_call`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-web_search_tool_call) | [`web_search_call`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-web_search_tool_call)<br>• Tool Call does not tell about the exact input that goes into the tool</br> <br>• Tool Call object does not have annotation for cited URLs they are available as a part of separate message</br> |
| Code Interpreter | [`code_interpreter_call`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-code_interpreter_tool_call) | [`code_interpreter_call.outputs`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-code_interpreter_tool_call-outputs)<br>• Tool Response and Tool Call represented within sample object but separated</br> |
| File Search | [`file_search_call`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-file_search_tool_call) | [`file_search_call.results`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-file_search_tool_call-results)<br>• Tool Response and Tool Call represented within sample object but separated</br> |
| MCP | [`mcp_call`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-mcp_tool_call) | [`mcp_call.output`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-mcp_tool_call-output) |
| Function Calling | [`function_call`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output-function_tool_call) | [`function_call_output`](https://platform.openai.com/docs/api-reference/responses/create#responses_create-input-input_item_list-item-function_tool_call_output) |

**Architecture Pattern**: Separate Objects - All tools use separate objects in the [`output[]`](https://platform.openai.com/docs/api-reference/responses/object#responses-object-output) array for both tool calls and their corresponding responses.

## Claude Server Tools - Tool Call and Response Representation

| Tool | Tool Call Object | Tool Response Object |
|------|------------------|----------------------|
| Web Search | Not available as server tool<br>• Available through MCP connectors | Not available as server tool<br>• Available through MCP connectors |
| Code Executor | [`server_tool_use`](https://platform.claude.com/docs/en/api/python/messages#server_tool_use)<br>• `name`: "bash_code_execution"<br>• `name`: "text_editor_code_execution"<br>• Equivalent to Code Interpreter | [`bash_code_execution_tool_result`](https://platform.claude.com/docs/en/agents-and-tools/tool-use/code-execution-tool#bash-command-response)<br>[`text_editor_code_execution_tool_result`](https://platform.claude.com/docs/en/agents-and-tools/tool-use/code-execution-tool#file-operation-responses)<br>• Contains execution output and file operation results |
| File Search | Not available as server tool<br>• Available through uploaded files via Files API<br>• Files processed in code execution environment | Not available as server tool<br>• File content accessible via [`container_upload`](https://platform.claude.com/docs/en/build-with-claude/files) content block |

## Gemini Built-in Tools - Tool Call and Response Representation

| Tool | Tool Call Object | Tool Response Object |
|------|------------------|----------------------|
| Web Search | Model automatically decides to search<br>• No separate call object | [`groundingMetadata`](https://ai.google.dev/api/generate-content#v1beta.GroundingMetadata) in response<br>• Contains [`webSearchQueries`](https://ai.google.dev/api/generate-content#v1beta.GroundingMetadata), [`groundingChunks`](https://ai.google.dev/api/generate-content#v1beta.GroundingChunk), [`groundingSupports`](https://ai.google.dev/api/generate-content#v1beta.GroundingSupport)<br>• Embedded within candidate response |
| Code Execution | Model automatically generates code<br>• No separate call object | [`Part`](https://ai.google.dev/api/caching#Content) objects in [`content`](https://ai.google.dev/api/caching#Content)<br>• [`executableCode`](https://ai.google.dev/api/caching#Part) with generated code<br>• [`codeExecutionResult`](https://ai.google.dev/api/caching#Part) with execution output |
| File Search | Model automatically searches files<br>• No separate call object | [`groundingMetadata`](https://ai.google.dev/api/generate-content#v1beta.GroundingMetadata) in response<br>• Contains [`retrievedContext`](https://ai.google.dev/api/generate-content#v1beta.RetrievedContext) with file content<br>• Citations in [`groundingSupports`](https://ai.google.dev/api/generate-content#v1beta.GroundingSupport) |
| MCP | Not available as built-in tool<br>• Available through SDK integration<br>• Manual MCP server integration possible | Not available as built-in tool<br>• Custom implementation required |
| Function Calling | [`functionCall`](https://ai.google.dev/api/caching#FunctionCall) in [`Part`](https://ai.google.dev/api/caching#Part)<br>• Contains function name and arguments | [`functionResponse`](https://ai.google.dev/api/caching#FunctionResponse) in [`Part`](https://ai.google.dev/api/caching#Part)<br>• Contains function execution results |

**Architecture Pattern**: Mixed Representation - Code execution uses [`Part`](https://ai.google.dev/api/caching#Part) objects within [`content`](https://ai.google.dev/api/caching#Content) (executableCode, codeExecutionResult), while search and file search use separate [`groundingMetadata`](https://ai.google.dev/api/generate-content#v1beta.GroundingMetadata) at the candidate level. Function calling uses dedicated `functionCall` and `functionResponse` parts.
